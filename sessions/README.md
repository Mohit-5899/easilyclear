# Sessions Log

Daily build logs for the Gemma Tutor hackathon project.

## Purpose

One file per working day. At the end of the 33-day build, reading these files in order reconstructs exactly what happened, what was decided, and why.

## Convention

- **Filename:** `YYYY-MM-DD.md` (one per day)
- **Length:** ~30–60 lines. Logs, not essays.
- **Structure:** See the template in `../CLAUDE.md` §2.2
- **When written:** Plan section at start of day, rest filled in as you go, finalized at end of day
- **Commit:** Each day's session doc is committed with that day's code changes
