## Evening research — PageIndex deep dive (2 parallel agents)

**Decisions locked:**

1. **PageIndex is VENDORED, not pip-installed.**
   - `pip install pageindex` installs a different package (VectifyAI's paid hosted SDK)
   - OSS repo hardcodes LiteLLM and has no LLM injection point
   - We vendor 3 files (`page_index.py`, `utils.py`, `retrieve.py`), edit ~40 lines in `utils.py` to route through our `LLMClient`
   - Default PDF parser PyPDF2 → hardcode PyMuPDF
   - `asyncio.gather` unlimited concurrency → add semaphore(5)
   - Full spec in `ARCHITECTURE.md §11`

2. **Retrieval is LLM tool-calling, not algorithmic tree search.**
   - PageIndex's `retrieve.py` is 137 lines exposing 3 tools: `get_document`, `get_document_structure`, `get_page_content`
   - We build our own 60-line agent loop in `backend/retrieval/pageindex_agent.py` using those tools via our `LLMClient`
   - Cost: 3–6 tool calls per query, 2–5 seconds per answer
   - Indexing cost: 100–250 LLM calls per book → <$5 for all 9 NCERT Geography books one-time

3. **Source whitelist — non-negotiable.** Only `ncert.nic.in` + `rajeduboard.rajasthan.gov.in`. Coaching mirrors BANNED. Eliminates 95% of promotional contamination at layer 1.

4. **Four-layer content cleaning pipeline** (ARCHITECTURE §10):
   - L1: source whitelist (reject non-official)
   - L2: regex pre-filter (strip URLs, coaching names, handles, watermarks, pagination, copyright)
   - L3: LLM cleanup pass (targeted — only for flagged pages)
   - L4: manual JSON review (grep for residual contamination, hand-edit in flat-file JSON — superpower of JSON storage)

5. **English PDFs only for MVP ingestion** — skips Devanagari extraction issues entirely. Gemma 4 still outputs Hindi answers via glossary-injected prompts (already validated today with real API).

6. **JSON schema carries provenance:** `source_url`, `source_authority`, `source_publisher`, `language`, `ingested_at`, `cleaned_at`, `cleanup_version`, `pageindex_version`, `llm_model_indexing` at the root of every textbook tree. Full schema in ARCHITECTURE §10.3.

7. **BM25 safety net wired in from Day 3** — ~50 lines, always-on second opinion. LLM traversal has no backtracking; one bad branch decision makes the correct leaf unreachable. BM25 is the cheap fallback.

8. **Go/no-go checkpoint on Day 6** — after bulk indexing all 9 books, if ≥7 of 9 trees look clean, commit to PageIndex. If not, fall back to hybrid = manual TOC + BM25 + LLM reranker (NOT vector RAG — embedder dependency kills offline story).

**Alternatives rejected (with reasons):**
- **Vector RAG (FAISS + bge-m3)** — 1.1 GB RAM on demo laptop, can't cite pages without custom chunk metadata, chunk boundaries destroy hierarchical signal
- **RAPTOR** — still needs embeddings for UMAP clustering, loses chapter identity
- **GraphRAG** — indexing 10–50× more expensive, overkill for 9 books
- **Pure BM25** — Hindi→English mismatch, score 2/5 on Hindi queries
