# Spec — Cross-Book Dedup (Day 23–24)

**Status**: draft · **Last updated**: 2026-05-01

## User story

After ingesting Springboard Rajasthan Geography (already done) and RBSE Class 11 Geography (Day 24), the system identifies overlapping leaves (e.g., both have an "Aravalli formation" leaf). The most authoritative version wins; the rest are marked `superseded_by: <winner_node_id>` in YAML frontmatter so retrieval doesn't surface duplicates.

## Goals

- Detect near-duplicate leaves across books with high precision (false-positive ≤5%)
- Idempotent: re-running dedup on the same set produces identical results
- Costs < $0.25 per book ingest
- Runs as Stage 6.5 (after title refine, before content fill)

## Non-goals

- Within-book dedup (already prevented by structural validator's overlap check)
- Merging duplicate leaves into one (just flag, keep both for citation traceability)
- Auto-removing the loser file (kept on disk, just filtered at retrieval)

## Algorithm (per research `2026-05-01-dedup.md`)

**Hybrid: embedding prefilter + LLM judge for grey zone.**

```
For each (leaf_a, leaf_b) where book_a != book_b:
    sim = cosine(embed(leaf_a.title + leaf_a.body[:500]),
                 embed(leaf_b.title + leaf_b.body[:500]))
    if sim < 0.80:
        skip                                    # different
    elif sim < 0.92:
        verdict = llm_judge(leaf_a, leaf_b)     # grey zone
        if verdict == "duplicate":
            mark_superseded(loser=lower_authority(a, b), winner=...)
    else:                                       # >= 0.92
        mark_superseded(loser=lower_authority(a, b), winner=...)
```

Embedding model: `BAAI/bge-small-en-v1.5` (33M params, 384-dim, runs on CPU in seconds).
LLM judge: Gemma 4 26B paid tier, prompt asks for `"duplicate" | "related" | "different"`.

## Winner-selection rule (deterministic)

1. **Source authority** (highest wins):
   - NCERT (rank 0)
   - RBSE (rank 1)
   - Springboard / coaching (rank 2)
   - other (rank 3)
2. **Longer body** (more content → more authoritative)
3. **Smaller node_id** lexicographic (stable tie-break)

Authority rank lives in `book_metadata.publisher_rank`. Set at ingest time.

## Output: superseded_by field

Existing leaves whose new ingest "wins" get a YAML frontmatter update on disk:

```yaml
---
name: Aravalli Mountain Range
node_id: geography/springboard_rajasthan_geography/.../aravali
superseded_by: geography/ncert_class11_geo/.../aravali  # ← NEW
superseded_at: 2026-05-08T14:23:01Z
superseded_reason: "duplicate per BGE+LLM judge (sim=0.94)"
---
```

If new leaf loses, the new node gets the field instead. Idempotent: re-running dedup with no new books is a no-op.

## Embedding cache

Per CLAUDE.md "use what we have": adapt the `content_hash` already in our YAML frontmatter as the cache key. Stores `{content_hash: list[float]}` in `database/embeddings_cache.json`.

```python
def get_embedding(text: str, model_name: str) -> list[float]:
    h = sha256(text.encode()).hexdigest()
    if h in cache:
        return cache[h]
    vec = model.encode(text)
    cache[h] = vec
    persist()
    return vec
```

## Backfill script

`scripts/dedup_existing.py` — runs dedup on already-ingested books. One-shot, idempotent. Used to dedup Springboard against itself or against future ingests.

## Success criteria

- Springboard Aravalli leaf vs RBSE Aravalli leaf → flagged duplicate, RBSE wins (rank 1 < rank 2)
- Springboard Climate leaf vs RBSE Soil leaf → flagged different (different topics)
- Re-run dedup with same inputs → identical `superseded_by` outputs
- Total runtime < 10 min for 2 books × ~30 leaves each = ~900 pairs (most filtered by embedding)

## Edge cases

- **Both leaves have same authority** (e.g., two NCERT books): tie-break by longer body, then node_id
- **Loser already has superseded_by** (chain): walk to root, supersede whichever is the active head
- **Network failure mid-judge**: pause + retry; partial progress saved per pair (no full restart)
