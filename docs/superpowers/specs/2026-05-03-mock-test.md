# Spec — Mock Test Generator (Day 21–22)

**Status**: draft · **Last updated**: 2026-05-01

## User story

A student selects the **Climate of Rajasthan** chapter in `/explorer`, opens the **Practice** tab, clicks "Generate 10 questions". The system returns 10 MCQs with answer keys grounded in the source paragraphs. She enters full-screen test mode, answers each, then sees a review screen showing correct/incorrect, the cited source paragraph for each, and a score.

## Goals

- 10 MCQs generated in < 30 seconds
- 100% of correct answers verifiable via deterministic span-match in source
- Distractors plausible but unambiguously wrong per source
- Mix of easy / medium / hard difficulty

## Non-goals

- Free-response (essay) questions
- Adaptive difficulty (just 4-3-3 split)
- Saved test history across sessions
- Cross-book questions

## Generation strategy (per research `2026-05-01-mcq-generation.md`)

3-stage pipeline:

### Stage 1: Schema-constrained generation
LLM emits JSON conforming to the schema below. Use Gemma 4 26B with `response_format={"type": "json_object"}`.

### Stage 2: Deterministic span verification
For each question, check that `answer_span` is a substring of one of the cited paragraphs. If not → reject and regenerate.

### Stage 3: LLM-judge single-correct check + leakage probe
Second LLM call (also Gemma 4 26B) with strict prompt: *"Given ONLY the paragraphs below, can a reader determine the correct answer? Is the answer the ONLY single correct option?"* If no → reject.

Reject rate budget: up to 30%. Generate 13 raw, keep top 10 that pass.

## Question schema

```json
{
  "id": "q-uuid",
  "prompt": "Which classification ranks Rajasthan as having the largest climate zone?",
  "choices": {
    "A": "Koppen",
    "B": "Trewartha",
    "C": "Thornthwaite",
    "D": "General Classification"
  },
  "correct": "A",
  "answer_span": "It is the largest climate zone according to Koppen",
  "source_node_id": "geography/.../03-climatic-phenomena-and-isohyets",
  "source_paragraph_ids": [142, 143],
  "difficulty": "medium",
  "bloom_level": "understand",
  "distractor_rationales": {
    "B": "Trewartha is mentioned but not as 'largest'.",
    "C": "Thornthwaite has 4 parts but no 'largest' claim.",
    "D": "General Classification is a parent category, not a peer."
  },
  "explanation": "Per the source, Steppe (semi-arid) is the largest Koppen climate zone. Other classifications don't make this claim."
}
```

## Generator prompt skeleton

```
You are an exam-prep question writer for the RAS Pre (Rajasthan
Administrative Services Preliminary) exam. Write {N} MCQs from the source
paragraphs below.

Rules:
1. Every correct answer MUST be supported by an exact substring of one of
   the source paragraphs (the answer_span).
2. Each question targets ONE specific fact, definition, classification,
   number, or named entity.
3. Distractors must be plausible (same category, similar surface form) but
   unambiguously wrong per the source.
4. NO "all of the above" / "none of the above" / negation stems.
5. NO trivia outside the source.
6. Mix difficulty: {easy_count} easy, {med_count} medium, {hard_count} hard.

Source paragraphs:
[para_id={pid} page={p}] {text}
[para_id={pid} page={p}] {text}
...

Output JSON:
{ "questions": [Question, ...] }
```

## Difficulty calibration

- **Easy**: lexical/obvious foils (e.g. distractors are unrelated entities). Stem mirrors source language.
- **Medium**: same-category foils (e.g. multiple climate zones, pick the right one). Stem requires understanding.
- **Hard**: same-category near-misses with similar numbers/dates/named entities, OR multi-hop within paragraph (combine 2 facts).

Avoid: hard-via-obscurity (asking about a footnote nobody would notice). Hard means cognitive load, not adversarial trivia.

## Success criteria

- Generate 10 MCQs from any leaf node with ≥5 paragraphs in < 30 seconds
- 100% of generated questions pass span verification (Stage 2)
- ≥95% pass LLM judge (Stage 3) on first try
- Manual review of 30 generated questions across 3 nodes: every "correct" is verifiable by a human reading the source

## Edge cases

- **Node has < 5 paragraphs**: return error "Not enough source content for a test"
- **Generator outputs invalid JSON**: regenerate up to 3 times, then fail with informative error
- **Span verification fails for a question**: drop that question, regenerate one more if below quota
- **All 13 candidates rejected**: return whatever passed (with a warning), don't block

## Testing (TDD)

- `test_question_schema.py` — Pydantic model accepts/rejects edge inputs
- `test_span_verifier.py` — exact substring, normalized whitespace, Unicode quotes; rejects substring not in any cited paragraph
- `test_test_orchestrator.py` — end-to-end on mock LLM client returning canned good/bad questions; verifies retry + final filter

## Open decisions

- **Embed difficulty selection in prompt vs post-filter**: → embed in prompt, audit on review.
- **Question UUID generation**: server-side (avoid client-spoofed IDs)
- **Time limit per test**: 15 minutes (UI countdown), no enforcement
